def stats(df):
    return pd.concat([df.nunique(), 
                      df.isnull().sum(),
                      df.dtypes], axis=1).rename({0: 'nunique', 1: 'nancount', 2:'dtype'}, axis=1)

Appunti:¶

  • PatientId non ha cardinalità uguale a quella del dataset, lo stesso paziente può prendere più appuntamenti. Forse si potrebbe usare come feature perché potrebbero esserci pazienti più inclini a non rispettare appuntamenti di altri a prescindere dalle altre features.

  • AppointmentID è univoco, si può scartare.

  • Entrambe le date andrebbero mantenute, eventualmente con qualche trasformazione. AppointmentDay ha solo 27 valori univoci.

  • Age, credo sia da usare ma prima ci sono da indagare eventuali correlazioni...

  • Neighbourhood potrebbe determinare il risultato se alcuni posti sono "fuori mano" ma sono 81 classi!... da rivedere.

  • Le restanti sono features categoriche che, nel momento in cui scrivo, terrei tutte.

# Prende un DataFrame e restituisce un DataFrame pronto per il preprocessing
def refine(df):
    
    # Copia il dataframe solo per sicurezza
    x = df.copy()    
        
    # Aggiunge la colonna dei giorni di distanza tra la richiesta e l'appuntamento
    # Nota: in caso di date NaN anche WaitingDay sarà NaN, questo si gestirà nel preprocessore.
    x['WaitingDays'] = (train_data.AppointmentDay.dt.date - train_data.ScheduledDay.dt.date).dt.days
        
    # Categorizza età
    x['Age'] = x.Age.apply(age_class)
    
    # Categorizza i giorni di attesa
    x['WaitingDays'] = x.WaitingDays.apply(wdays_class)
    
    # Feature selection
    x.drop(['AppointmentID', 'PatientId', 'ScheduledDay', 'AppointmentDay'], axis=1, inplace=True)
            
    return x

def model_builder(input_size, layer_sizes):
    
    inputs = keras.Input(shape=(input_size,))
    x = layers.Dense(layer_sizes[0], activation="relu")(inputs)
    
    # Aggiungi layers 
    for size in layer_sizes[1:]:
        x = layers.Dense(size, activation="relu")(x)
        x = layers.Dropout(0.3)(x)
        
    outputs = layers.Dense(1, activation='sigmoid')(x)
    
    model = keras.Model(inputs=inputs, outputs=outputs, name="model")
    
    model.compile(loss='mse', optimizer='adam', metrics=["accuracy"])
    
    return model

Bilanciamento del dataset¶

Negli addestramenti precedenti si ottiene l'80% fisso, tuttavia questo succede perché la maggior parte dei pazienti si presenta all'appuntamento e come si vede dalla heat map questo falsa i risultati. In questa parte, addestro lo stesso modello bilanciando il dataset in modo tale che ci siano numeri equivalenti di label.

# Bilancia i dati in modo che ci siano tante label di una classe quante dell'altra
def balance(df):
    num_noshow = df[train_data['No-show'] == 'Yes'].shape[0]
    num_show = df.shape[0] - num_noshow
    
    if num_noshow > num_show:
        data_show = df.loc[df['No-show'] == 'No']
        data_noshow = df.loc[df['No-show'] == 'Yes'].sample(n=num_show)
    else:
        data_noshow = df.loc[df['No-show'] == 'Yes']
        data_show = df.loc[df['No-show'] == 'No'].sample(n=num_noshow)
    
    return pd.concat([data_show, data_noshow], 
                     axis=0, 
                     ignore_index=True).sample(frac=1, random_state=42).reset_index(drop=True)
